پروژه لیست کارهای روزمره (Todo App) با Express.js

این یک پروژه وب سرور ساده برای مدیریت لیست کارهای روزمره (Todo) است که با استفاده از Express.js در محیط Node.js ساخته شده است. این پروژه یک API پایه RESTful را برای عملیات‌های CRUD (ایجاد، خواندن، به‌روزرسانی، حذف) پیاده‌سازی می‌کند.

تمام داده‌ها به صورت موقت در حافظه (in-memory) و در یک آرایه ذخیره می‌شوند و با راه‌اندازی مجدد سرور، پاک خواهند شد.

ویژگی‌ها

API RESTful: شامل اندپوینت‌هایی (endpoints) برای عملیات‌های GET, POST, PUT, و DELETE.

سرو فایل‌های استاتیک: سرو کردن یک فرانت‌اند ساده (HTML/CSS/JS) از پوشه‌ی public.

میان‌افزار (Middleware): استفاده از میان‌افزار express.json() برای پارس کردن بدنه‌ی درخواست‌های JSON.

مدیریت خطای پایه: شامل یک کنترل‌کننده‌ی ۴۰۴ برای مسیرهای یافت‌نشده.

ساختار پوشه‌ها

todo-app/
├── public/
│   ├── index.html     (فرانت‌اند اپلیکیشن)
│   ├── style.css      (استایل‌های CSS)
│   └── script.js      (کدهای جاوااسکریپت فرانت‌اند)
├── app.js             (فایل اصلی سرور Express)
├── package.json       (تنظیمات پروژه و وابستگی‌ها)
└── README.md          (همین فایل)


نحوه راه‌اندازی و اجرا

برای اجرای این پروژه به صورت محلی، مراحل زیر را دنبال کنید:

نصب وابستگی‌ها:
ابتدا باید Express.js را نصب کنید. در ترمینال و در پوشه‌ی اصلی پروژه، دستور زیر را اجرا کنید:

npm install


اجرای سرور:
برای راه‌اندازی سرور، از دستور زیر استفاده کنید:

npm start


(این دستور، اسکریپت node app.js را که در package.json تعریف شده است، اجرا می‌کند.)

مشاهده اپلیکیشن:
پس از اجرا، سرور روی پورت ۳۰۰۰ در دسترس خواهد بود.
مرورگر خود را باز کنید و به آدرس http://localhost:3000 بروید تا اپلیکیشن را ببینید.

اندپوینت‌های API (API Endpoints)

تمامی اندپوینت‌های API در مسیر /api/todos در دسترس هستند.

متد (Method)

مسیر (Route)

توضیحات

GET

/api/todos

دریافت لیست تمام کارها.

POST

/api/todos

ایجاد یک کار جدید. بدنه‌ی درخواست باید شامل {"text": "..."} باشد.

PUT

/api/todos/:id

تغییر وضعیت (انجام‌شده/نشده) یک کار با id مشخص.

DELETE

/api/todos/:id

حذف یک کار با id مشخص.
